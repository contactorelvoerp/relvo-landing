# Flujo recomendado para empezar

Esta página resume el recorrido sugerido para implementar Relvo por primera vez. Te sirve para avanzar en el orden correcto y evitar tener que volver atrás para completar información clave.

## Antes de comenzar

Si tu organización recién está empezando a usar Relvo, sigue este recorrido como secuencia base. El objetivo es dejar primero las reglas globales y luego cargar la información operativa que depende de ellas.

## Recorrido recomendado de implementación

1. Empieza en [Configuración](../configuracion/overview.md) para revisar preferencias de facturación y aprobaciones.
2. Sigue en [Pricing](../pricing/overview.md) para cargar items, métricas y planes.
3. Pasa a [Clientes](../clientes/overview.md) para crear clientes y sus cuentas de facturación.
4. Continúa en [Contratos](../contratos/overview.md) para dar de alta acuerdos comerciales.
5. Usa [Uso](../uso/overview.md) para registrar eventos o importar cargas en lote.
6. Revisa [Facturación](../facturacion/overview.md) para validar pre-facturas, emitir y seguir documentos.
7. Cierra el circuito en [Aprobaciones](../aprobaciones/overview.md) y [Notificaciones](../notificaciones/overview.md) para resolver pendientes operativos.

## Recorrido corto para capacitación

Si vas a entrenar a un usuario nuevo, este orden funciona bien para mostrar la aplicación de lo general a lo operativo:

1. Panel de Ingresos
2. Clientes
3. Detalle de cliente
4. Contratos
5. Detalle de contrato
6. Uso
7. Facturación
8. Aprobaciones
9. Configuración
